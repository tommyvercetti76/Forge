# Book Localization Audit and Agent Handoff

Status: draft handoff  
Goal owner: Forge book-to-subtitle/audio workflow  
Target output languages: English, Hindi, Marathi

## Goal

Build and audit a production-ready workflow that takes exactly 10 pages from a given book and produces near-perfect English, Hindi, and Marathi subtitle text plus matching narration audio for long-form videos.

For this workflow, "done" does not mean "the pipeline produced files." Done means the selected source pages are fully represented, translations are reviewed, subtitles use approved text, audio matches those subtitles, and the manifest proves source-to-output lineage.

## What Must Be Produced

For a selected 10-page range, Forge must produce:

- Canonical source text for the selected pages.
- Segment-level English text.
- Segment-level Hindi translation.
- Segment-level Marathi translation.
- Reviewer-facing translation sheet.
- Approved subtitle files for English, Hindi, and Marathi.
- Narration audio for English, Hindi, and Marathi.
- Optional muxed videos using the produced audio/subtitles.
- A manifest linking every source segment to translations, subtitles, audio, review status, and QC results.

## Current Implementation Map

Relevant files:

- `bin/audiobook.py`
  - Main ASMR audiobook/video pipeline.
  - Accepts `--langs en,hi,mr`.
  - Accepts `.rtf`, `.txt`, `.md`, and `.pdf`.
  - Has `--batch-pages`, `--page-words`, and `--spoken-words`.
  - Uses Sarvam TTS for Hindi/Marathi and Kokoro/Sarvam for English.
  - Generates subtitles from mastered audio via `mlx_whisper`.

- `bin/forge_runtime.py`
  - Contains `translate_texts_ollama`.
  - Default translation model is `hf.co/mradermacher/sarvam-translate-GGUF:Q4_K_M`.
  - Handles retry/fallback behavior for placeholder translation failures.

- `bin/forge.py`
  - Has stronger translation QA logic in `translation_qc_twice`.
  - The stronger logic is used by episode/audiobook paths in `forge.py`, but not fully wired into `bin/audiobook.py` ASMR mode.

- `bin/forge_web.py`
  - Exposes audiobook and ASMR audiobook tabs.
  - Does not currently make the "full 10 pages versus excerpt from 10-page window" distinction clear enough.

## Critical Gaps

### P0: 10 Pages Does Not Currently Mean Full 10 Pages

In `bin/audiobook.py`, `--batch-pages 10` defines a 10-page source window, but `--spoken-words` defaults to `150`.

That means a 10-page batch currently speaks/translates only the first 150 words from the 10-page window unless explicitly overridden. This is not acceptable for full 10-page localization.

Required outcome:

- Add a clear full-page mode.
- The user must be able to select pages 1-10 and get all text from those pages.
- The dry-run manifest must show exact source word count, selected page range, and produced segment count.

### P0: Subtitles Are Not Canonical Approved Translation Text

Current ASMR subtitles are generated by Whisper from mastered audio. This is useful for timing, but it makes Whisper transcription the subtitle text.

For near-perfect translation, approved translation text must be the subtitle source of truth. Whisper or alignment may be used to derive timings, but not to silently rewrite the subtitles.

Required outcome:

- Approved translation text is canonical.
- Subtitle files are built from approved segment text.
- Whisper/ASR output is used only for alignment verification or mismatch detection.

### P0: No Human Review Gate

Near-perfect Hindi and Marathi translation requires review. The pipeline currently has no hard "approved" state per segment.

Required outcome:

- Generate a review sheet.
- Each segment has approval status.
- Final production output cannot be marked complete until all required segments are approved or explicitly waived.

### P0: No Glossary Or Style Guide Enforcement

Names, places, cultural terms, honorifics, idioms, numbers, dates, and recurring terms must be consistent across English, Hindi, and Marathi.

Required outcome:

- Add glossary input support.
- Add style guide input support.
- QC checks that glossary terms are preserved or translated according to policy.

### P0: No Source Extraction Audit

PDF/RTF extraction can drop paragraphs, page breaks, footnotes, headers, or punctuation. Current extraction produces plain text but does not prove that exactly 10 pages were captured.

Required outcome:

- Record page boundaries.
- Record raw extracted text per page.
- Flag empty pages, very short pages, OCR garbage, repeated headers/footers, and suspicious character loss.

### P1: Translation QA Is Too Weak In ASMR Path

`bin/forge.py` has `translation_qc_twice`, which translates twice and back-translates. `bin/audiobook.py` currently translates whole batches once in the ASMR path.

Required outcome:

- Add multi-pass translation QA to the ASMR/book localization path.
- Store backtranslations and scores in the manifest.
- Flag likely omissions, hallucinations, and bad length ratios.

### P1: Subtitle Readability Is Not Gated

The current pipeline can emit subtitle blocks without enforcing production readability.

Required outcome:

- Max two lines per cue.
- Language-specific max characters per line.
- Reasonable reading speed.
- No empty cues.
- No overlapping cues.
- Minimum and maximum cue duration.

Suggested thresholds:

- English: max 42 chars per line.
- Hindi: max 34 chars per line.
- Marathi: max 34 chars per line.
- Max two lines per cue.
- Target reading speed: 12-18 chars/sec, hard fail above 22 chars/sec.
- Minimum cue duration: 0.8 sec.
- Maximum cue duration: 7 sec.
- Minimum gap between cues: 80 ms when possible.

### P1: VTT Output Is Not Truly VTT

The ASMR pipeline exposes `--subtitles vtt`, but the writer path appears SRT-oriented. VTT requires `WEBVTT` header and dot timestamps.

Required outcome:

- Either remove VTT from UI until implemented, or add a real VTT writer.

### P1: Audio QC Is Incomplete

Audio is generated and mastered, but pass/fail QC is not strict enough for production.

Required outcome:

- Detect silent or near-silent TTS chunks.
- Detect clipping.
- Record loudness target.
- Detect duration drift.
- Detect wrong-language ASR output.
- Verify final audio duration matches subtitle coverage.

Suggested thresholds:

- Peak must be below -1.0 dBFS.
- Integrated loudness target around -19 LUFS for web narration, or explicitly configured.
- No silent segment longer than 2 seconds unless present in subtitle timing.
- Subtitle coverage should span at least 98 percent of spoken audio duration.
- Final muxed video duration should match audio within 0.5 sec.

### P2: Web UI Does Not Express The Real Workflow

The current UI exposes many audiobook controls, but it does not make these production concepts explicit:

- Page range.
- Full selected pages versus excerpt.
- Translation review required.
- Glossary/style guide.
- Approved subtitles versus Whisper transcription.
- QC pass/fail status.

Required outcome:

- Add a dedicated "Book Localization" workflow or simplify ASMR audiobook for this use case.
- Hide non-essential thumbnail/ASMR controls until after translation/audio is approved.

## Target Pipeline

### Step 1: Ingest

Inputs:

- Book file: `.pdf`, `.rtf`, `.txt`, or `.md`.
- Page range: exactly 10 pages, for example `1-10`.
- Languages: `en,hi,mr`.
- Optional glossary file.
- Optional style guide file.
- Optional base video for muxing.

Artifacts:

- `source/pages.json`
- `source/pages/page-001.txt` through `source/pages/page-010.txt`
- `source/source.en.txt`
- `source/source_audit.json`

Done:

- Exactly 10 pages extracted.
- Every selected page has non-empty text unless explicitly marked blank.
- Page order is preserved.
- Headers/footers are either removed or logged.
- Source word count is recorded.

### Step 2: Segment

Split the source into stable translation units.

Preferred unit:

- Paragraph when paragraph is short enough.
- Sentence when paragraph is too long.
- Never split names, quotes, or dialogue awkwardly if avoidable.

Artifacts:

- `segments/segments.en.json`

Each segment must include:

- `segment_id`
- `page_number`
- `paragraph_index`
- `sentence_index`
- `source_text`
- `source_word_count`
- `source_char_count`

Done:

- 100 percent of source text maps to segment IDs.
- Segment IDs are stable across reruns.
- No segment is empty.

### Step 3: Translate

Translate each segment into Hindi and Marathi.

Artifacts:

- `translations/translations.en.json`
- `translations/translations.hi.json`
- `translations/translations.mr.json`

Each translated segment must include:

- `segment_id`
- `source_text`
- `translated_text`
- `language`
- `model`
- `prompt_version`
- `glossary_version`

Done:

- Every English segment has exactly one Hindi segment.
- Every English segment has exactly one Marathi segment.
- Segment count matches across all languages.
- No target segment equals source text unless explicitly allowed.

### Step 4: Translation QA

Run automatic QA before human review.

Checks:

- Backtranslation to English.
- Named entity preservation.
- Number/date preservation.
- Glossary compliance.
- Length ratio outliers.
- Placeholder or untranslated text.
- Script detection.
- Punctuation and quote sanity.
- Omission or hallucination risk.

Artifacts:

- `qc/translation_qc.json`
- `review/review_sheet.csv`

Done:

- All P0 translation QC failures are resolved.
- P1 warnings are either fixed or accepted with notes.
- Human reviewer can inspect source, translation, backtranslation, and issues side by side.

### Step 5: Human Review

The reviewer approves or edits each Hindi and Marathi segment.

Artifacts:

- `review/review_sheet.csv`
- `translations/approved.hi.json`
- `translations/approved.mr.json`
- `translations/approved.en.json`

Done:

- Every required segment has `approved=true`.
- Every edited segment preserves its original `segment_id`.
- Reviewer name/date is recorded.
- Review status is written to manifest.

### Step 6: Audio Generation

Use approved text as the only input to TTS.

Recommended engines:

- English: Kokoro or Sarvam English, depending preferred voice.
- Hindi: Sarvam Bulbul.
- Marathi: Sarvam Bulbul with Marathi-specific speaker override when needed.

Artifacts:

- `audio/raw/book.en.raw.wav`
- `audio/raw/book.hi.raw.wav`
- `audio/raw/book.mr.raw.wav`
- `audio/master/book.en.master.wav`
- `audio/master/book.hi.master.wav`
- `audio/master/book.mr.master.wav`

Done:

- No failed or silent chunks.
- Audio passes loudness and clipping checks.
- Audio text input exactly matches approved translation text.
- TTS engine, speaker, pace, and sample rate are recorded.

### Step 7: Subtitle Timing

Subtitles must use approved text.

Timing options:

- Preferred: forced alignment against approved text.
- Acceptable interim: proportional timing from TTS segment durations.
- Verification: Whisper transcription compared against approved text.

Artifacts:

- `subtitles/book.en.srt`
- `subtitles/book.hi.srt`
- `subtitles/book.mr.srt`
- Optional real VTT files if implemented.
- `qc/subtitle_qc.json`

Done:

- Subtitle text exactly matches approved text after allowed formatting normalization.
- Timing covers the full audio.
- No overlapping cues.
- Readability thresholds pass.
- Hindi and Marathi text are not replaced by Whisper paraphrases.

### Step 8: Optional Video Mux

If a base video is provided, mux each language audio and subtitle track.

Artifacts:

- `video/book.en.mp4`
- `video/book.hi.mp4`
- `video/book.mr.mp4`

Done:

- Video duration matches audio within 0.5 sec.
- Audio track is present.
- Subtitle sidecar or embedded subtitle is present, depending configured output.
- Manifest records mux settings.

### Step 9: Final Manifest

Artifact:

- `localization_manifest.json`

Manifest must include:

- Book path.
- Page range.
- Source extraction summary.
- Segment counts.
- Translation model and prompt versions.
- Glossary/style guide paths.
- Reviewer approval summary.
- Audio file paths.
- Subtitle file paths.
- Video file paths.
- QC report paths.
- Overall pass/fail.

Done:

- Manifest can be used to reproduce and audit the entire job.
- Overall status is `PASS` only when all required checks pass.

## Proposed CLI

Preferred new command:

```bash
python3 bin/forge.py book-localize \
  --book /path/to/book.pdf \
  --pages 1-10 \
  --langs en,hi,mr \
  --glossary brand/localization/glossary.json \
  --style-guide brand/localization/style-guide.md \
  --video /path/to/base-video.mp4 \
  --out ~/Desktop/forge-test/book-localization \
  --review-required
```

Acceptable interim command if implemented through `bin/audiobook.py`:

```bash
python3 bin/audiobook.py \
  --rtf /path/to/book.pdf \
  --video /path/to/base-video.mp4 \
  --out-dir ~/Desktop/forge-test/book-localization \
  --langs en,hi,mr \
  --batch-pages 10 \
  --full-pages \
  --subtitles srt \
  --bed none \
  --mode normal
```

Important: do not rely on `--spoken-words 150` for this use case. That creates an excerpt, not full 10-page localization.

## Agent Work Plan

### Phase 1: Audit Current Behavior

Tasks:

- Run a dry-run with a representative 10-page source.
- Confirm whether current `--batch-pages 10` localizes full pages or only `spoken_words`.
- Inspect generated `manifest.json`.
- Verify whether subtitles are created from Whisper text or approved translation text.
- Confirm VTT output format if `--subtitles vtt` is used.

Done:

- Findings recorded in `docs/BOOK_LOCALIZATION_AUDIT_RESULTS.md`.
- Every P0 gap has file/line references.

### Phase 2: Add Full 10-Page Mode

Tasks:

- Add page range support if the source format can provide pages.
- Add `--full-pages` or equivalent.
- Ensure `spoken_words` cannot silently truncate full-page mode.
- Update web UI labels to distinguish excerpt mode from full localization mode.

Done:

- A 10-page input produces all text from the selected pages.
- Dry-run manifest proves coverage.
- Tests cover default excerpt mode and full-page mode.

### Phase 3: Add Translation QA And Review Sheet

Tasks:

- Reuse or move `translation_qc_twice` into shared runtime code.
- Add glossary/style guide support.
- Generate review CSV.
- Add approval state.

Done:

- Hindi and Marathi translations have backtranslations.
- Review sheet is generated.
- Pipeline can stop before audio if review is required.

### Phase 4: Make Approved Text Canonical For Subtitles

Tasks:

- Generate SRT from approved segment text.
- Use alignment or segment duration for timing.
- Use Whisper only as verification.
- Add real VTT writer or remove VTT option.

Done:

- SRT text matches approved translation text.
- Whisper mismatch report is generated.
- Subtitle readability QC passes.

### Phase 5: Add Audio QC

Tasks:

- Detect silent chunks.
- Detect clipping.
- Record duration, sample rate, peak, and loudness.
- Validate audio/subtitle coverage.

Done:

- Audio QC report exists for all languages.
- Any failed audio chunk blocks final PASS.

### Phase 6: Web UI Cleanup

Tasks:

- Add a dedicated Book Localization workflow.
- Show only required fields first: book, pages, languages, review required, video, output.
- Move ASMR bed, thumbnail, speaker details, and advanced settings into collapsed sections.

Done:

- A non-technical user can run a dry-run without understanding `spoken_words`.
- The UI never implies that excerpt mode is full 10-page translation.

## Final Definition Of Done

The project is done when all of the following are true:

- User can select exactly 10 pages from a book.
- Source extraction audit proves those 10 pages were captured.
- English, Hindi, and Marathi segment counts match.
- Every Hindi and Marathi segment has automatic QA.
- Every Hindi and Marathi segment has human approval or explicit waiver.
- Glossary and style guide rules are enforced or reported.
- Final subtitles use approved text, not raw Whisper transcription.
- Subtitle readability QC passes for all three languages.
- Audio is generated from approved text.
- Audio QC passes for all three languages.
- Optional muxed videos pass duration and track checks.
- Final manifest records every artifact and marks the job PASS.
